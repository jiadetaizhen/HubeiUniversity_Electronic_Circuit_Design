# 电子线路课程设计 – 基于STM32F103C8T6的多功能开发板

## 项目概述

本项目为电子信息学院《电子线路设计》课程的综合实训课题，面向电子信息工程专业本科生。课题要求学生**自主完成**电路原理图设计、PCB绘制、元器件焊接、硬件调试，并基于[STM32CubeMX](https://www.st.com/en/development-tools/stm32cubemx.html)和HAL库编写嵌入式固件程序，最终实现一个具备显示、交互、传感、时钟和信号采集等多功能的小型嵌入式系统。

通过这一完整的项目实践，学生将深入理解Cortex‑M3内核微控制器的工作原理，掌握常用数字接口（SPI、I2C、USART、单总线、并行总线等）的驱动方法，并学会将逻辑电路（译码器、锁存器）与MCU结合，解决实际工程中的IO扩展问题。

## 硬件设计详解

### 1. 主控与时钟系统
STM32F103C8T6支持外部高速晶振（HSE）和外部低速晶振（LSE）。本设计采用 **8MHz无源晶振** 作为HSE，经PLL倍频至 **72MHz** 系统时钟（SYSCLK）。AHB总线为72MHz，APB1为36MHz（最大允许），APB2为72MHz。  
- **PD0** – HSE振荡器输入（接晶振及负载电容）  
- **PD1** – HSE振荡器输出  
- **PC14/PC15** – LSE 32.768kHz晶振（本设计未使用内部RTC）

> **注意**：PD0/PD1/PC14/PC15均为专用引脚，不可作为普通GPIO使用，在CubeMX配置中必须正确设置为时钟功能。

### 3. 数码管驱动逻辑（重点）
两个**共阳极**三位数码管（共6位）采用**动态扫描**方式显示。由于STM32F103C8T6的GPIO数量有限（48引脚），无法直接驱动6×8=48个段位，因此采用 **HC573 8位锁存器** 输出段码， **74HC138 3-8译码器** 进行位选。

- **段码数据总线**：HC573的D0~D7分别连接至MCU的PB0、PB12、PB13、PB14、PB15、PA8、PA11、PA12（共8个GPIO）。MCU先将要显示的段码（共阳极数码管段码为低电平点亮）输出到这些IO上，然后给HC573的LE引脚（由MCU另一个GPIO控制，本设计未列明，但可复用其他空闲IO）一个高脉冲，将数据锁存到输出端。
- **位选控制**：74HC138的A0-A2由PB3~PB5控制，译码输出Y0-Y7（低电平有效）。将6位数码管的公共端分别接至Y0-Y5，MCU通过改变A2-A0的二进制值（0-5）选中当前要点亮的那一位。

### 4. 其他外设硬件连接
- **TFT LCD**：1.44寸ST7735S，支持SPI模式。除SPI四线（SCK、MOSI、CS、DC）外，还需RES复位线。本设计使用硬件SPI1，最高速率可达18MHz。
- **OLED**：0.96寸SSD1306，I2C接口（7位地址0x3C或0x3D）。由于硬件I2C1已被RX8025占用，I2C2被ADS1115占用，故OLED使用GPIO模拟I2C时序，以节省硬件资源。
- **DHT11**：单总线协议，对时序要求严格（微秒级）。需外接4.7kΩ上拉电阻，MCU通过PA15进行读写。
- **RX8025**：I2C实时时钟芯片，内置温度补偿晶振，精度可达±5ppm。支持闹钟和定时中断功能。本设计使用硬件I2C1（PB6/PB7）通信，地址为0x64（写）/0x65（读）。
- **ADS1115**：16位Δ-Σ ADC，I2C接口，可配置为单端或差分输入，可编程增益（PGA），采样率最高860SPS。使用硬件I2C2（PB8/PB9），地址默认为0x48（可通过ADDR引脚配置）。
- **CH340G**：USB转UART芯片，将USART1（PA9/PA10）转换为USB虚拟串口，用于PC串口信息调试。

### 3. 外设驱动实现要点

#### （1）SPI（TFT LCD）
- 使用SPI1，配置为主机模式，时钟极性（CPOL）和相位（CPHA）依据ST7735S数据手册设置为0（模式0）或3（模式3），本设计选用模式0。
- 传输速率：分频系数设置为4，得到18MHz（72MHz/4），满足屏幕要求。
- 数据传输：采用HAL_SPI_Transmit()函数，对于大批量图像数据可启用DMA（本设计未使用）。
- 关键命令：初始化序列（软复位、退出睡眠、设置颜色模式、显示开启等）需严格参考手册。

#### （2）IIC（RX8025和ADS1115）
- 硬件I2C1和I2C2分别配置，使用HAL_I2C_Master_Transmit/Receive()进行读写。
- 注意RX8025的寄存器地址为8位，写入时先发送寄存器地址再跟数据；读取时先写寄存器地址，再启动接收。
- ADS1115配置寄存器（0x01）需设置通道、增益、连续/单次模式、采样率等。

#### （3）单总线（DHT11）
- 使用GPIO输出模式发送起始信号（拉低至少18ms，然后拉高20~40us），再切换为输入模式读取响应。
- 时序采用 **微秒级精确延时**，通过SysTick或简单的循环延时（需根据编译器优化调整）。建议使用__NOP()或硬件定时器实现。
- 数据校验：读取40位数据（湿度整数+小数+温度整数+小数+校验和），校验和必须等于前4字节之和的低8位。

#### （4）数码管动态扫描
- ~~使用TIM2产生周期为 **1ms** 的中断（计数频率72MHz，预分频7200-1，自动重载10-1）。~~
- 在中断服务函数中执行刷新：先关闭所有位（将138的地址设为无效值，或直接给译码器输出高电平无效），然后根据显示缓冲区内容更新段码（向HC573写数据并锁存），最后设置138地址选中下一位。

### 4. 系统软件架构与任务调度

本设计采用 **FreeRTOS 实时操作系统** 进行多任务管理与调度，以取代传统的裸机前后台轮询模式。通过合理划分任务优先级，确保系统对显示刷新、人机交互、传感器数据采集等事件的实时响应能力。

系统共划分为以下几个核心任务，由 FreeRTOS 内核根据优先级进行抢占式调度：

1. **LED 数码管显示任务**：该任务每隔 5ms 执行一次，负责动态扫描刷新 6 位数码管。为保证显示稳定无闪烁，通过 `vTaskDelayUntil` 实现精准的固定周期调度。
2. **按键扫描任务**：每 20ms 执行一次，负责检测按键状态并进行软件消抖，将有效的按键事件（按下/长按/释放）通过消息队列发送至主控任务。
3. **主控任务（菜单与逻辑处理）**：阻塞等待按键消息队列，当接收到有效按键事件后，负责更新菜单界面、调整参数并刷新显示缓冲区。该任务优先级较低，仅在事件触发时运行。
4. **传感器采集任务**：以 2Hz 频率周期执行，负责启动 ADS1115 进行 ADC 转换、读取 DHT11 温湿度数据（单总线协议）以及读写 RX8025 时钟芯片，并将采集结果更新至全局共享变量中（配合互斥信号量保护）。

此外，FreeRTOS 的 **定时器服务任务** 可用于实现软件定时器功能（例如超时退出菜单、周期性蜂鸣器鸣响提示等）。

**系统同步与通信机制**：
- **消息队列**：用于按键任务向主控任务传递按键事件。
- **互斥信号量（Mutex）**：用于保护传感器数据的共享资源，防止任务间同时访问造成数据错乱。
- **二值信号量**：可扩展用于 ADS1115 转换完成中断与采集任务的同步（若启用中断方式）。

**配置要点**：
- 系统时钟节拍（Tick）设置为 1000Hz（即 1ms 一个 Tick）。
- 分配足够的堆栈空间给每个任务（通常 128~256 字），并根据实际运行情况进行优化调整。
- 空闲任务（Idle Task）中可添加低功耗模式（如 WFI 指令），降低系统整体功耗。

### 5. 菜单与界面设计

本系统配备 **0.96寸OLED**（I2C接口）作为菜单交互界面，同时 **1.44寸TFT彩屏**（SPI接口）作为数据可视化界面，二者分工明确，协同工作。

#### 5.1 显示分工
- **OLED（菜单屏）**：负责显示多级菜单结构、参数设置项、用户交互提示。分辨率为128×64，可清晰展示4行文字。
- **TFT（数据屏）**：负责实时显示传感器数据曲线，包括温度曲线、湿度曲线和ADC采样电压曲线。分辨率为128×128，支持彩色绘图，直观反映数据变化趋势。

#### 5.2 OLED 多级菜单结构

菜单采用 **树形结构**，通过四个按键（上/下/确认/返回）进行导航。主界面默认显示时间、温湿度等综合信息，按“确认”键进入主菜单。

| **系统信息** | 显示固件版本、堆栈使用、运行时间等 | 只读信息，按“返回”退出 |

#### 5.3 TFT 数据可视化

TFT屏幕以图形化方式展示传感器历史数据，界面设计如下：

- **坐标轴**：X轴为时间（采样点序号，最多显示60个点），Y轴为量程（温度0~50℃，湿度0~100%，电压0~5V）。
- **曲线绘制**：
  - 温度曲线：红色线条
  - 湿度曲线：蓝色线条
  - ADC电压曲线：绿色线条
- **实时数值显示**：在曲线图下方或右上角，以数字方式显示当前最新值。
- **刷新率**：每2秒采样一次并更新曲线（由传感器采集任务触发）。

**TFT 显示示例布局**：
```
┌───────────────────────┐
│                       │ 
│                       │
│                       │
│                       │
│                       │
│                       │
│                       │
│                       │
└───────────────────────┘
```

#### 5.4 交互逻辑流程

1. **上电默认**：OLED显示主界面，TFT显示实时曲线（默认全部显示）。
2. **按键操作**：
   - 在主界面下，按“确认”进入主菜单；按“上/下”无反应（保持主界面）。
   - 在菜单各级中，“上/下”移动光标或调整数值；“确认”进入子项或保存设置；“返回”退回上一级（在顶层菜单时返回主界面）。
3. **数值修改**：在时间设置或数码管自定义数值界面，光标所在位闪烁，“上/下”调整（0~9循环），按“确认”移到下一位，全部位调整完后弹出确认保存提示。
4. **TFT控制**：通过“显示切换”菜单可关闭曲线显示（省电）或切换曲线类型，设置即时生效。

#### 5.5 任务间通信

- 菜单操作产生的设置变更（如RTC时间、蜂鸣器启用标志、数码管显示模式）通过 **全局变量** 或 **消息队列** 传递给相应任务。
- 为保证数据一致性，对共享资源（如RTC时间、数码管显示缓冲区）的访问使用 **互斥信号量** 保护。

## 快速上手指南

### 硬件准备
1. 根据 `01-Hardware-Design/` 中的原理图、PCB布局文件（Altium或立创EDA）进行电路板原理图绘制、PCB Layout和焊接。建议先焊接电源部分（DC-DC、3.3V稳压电路），确认输出电压正常后再焊接MCU和其他外设。
2. 焊接完成后，**用万用表检查电源与地之间是否有短路**，尤其注意MCU各VDD和VSS引脚,主板预留有Test-point，用于检查电压是否符合正常工作范围。
3. 连接ST-link、J-link、CMSIS-DAP至SWD接口（PA13/PA14）。
4. 连接USB线至CH340G，需要安装驱动，点击下方图标下载：

3. 打开Keil工程，编译（F7）。若出现错误，检查是否包含正确的头文件路径和宏定义（如 `USE_HAL_DRIVER`, `STM32F103C8Tx`）。

### 功能验证步骤
- **开机**：数码管显示“00.00”（或默认值），TFT显示欢迎界面（课程名+组号）。
- **按键测试**：按下“上键”应切换菜单项，屏幕同步更新。
- **温度读取**：连接DHT11（注意电源和地），等待2秒后，屏幕显示当前温湿度。
- **时钟读取**：需先设置时间（通过菜单），然后掉电再上电，检查时间是否继续走（RX8025内置电池）。
- **ADC测试**：调节电位器，可以查看电压值的变化。

### 常见故障排查
| 现象                   | 可能原因                           | 解决方法                                 |
|------------------------|------------------------------------|------------------------------------------|
| 程序无法下载           | SWD接口被占用或芯片焊接不良            | 检查PA13/PA14连接，按住复位键再下载      |
| 数码管不亮或无显示     | 段码或位选GPIO配置错误，HC573未使能 | 检查锁存器LE引脚电平，确保定时器中断开启 |
| TFT白屏               | SPI速率过高、初始化序列错误         | 降低SPI分频系数，核对初始化命令          |
| 串口无输出             | 波特率不匹配，CH340驱动未安装       | 重新安装驱动，检查USART1时钟使能         |
| DHT11无数据           | 上拉电阻缺失，时序延时不准          | 添加4.7k上拉，调整延时（使用示波器测波形）|

## 课程设计目标（分阶段）

### 硬件设计阶段
1. **理解STM32F103C8T6的引脚功能与电气特性**：包括IO口模式（推挽/开漏）、复用功能、5V兼容性等。
2. **掌握原理图绘制方法**：使用EDA工具（如Altium Designer、立创EDA）绘制完整原理图，规范标注网络名、元件值。
3. **掌握PCB布局布线规范**：遵循高频信号线短、电源去耦、晶振靠近MCU、地线完整等原则。
4. **熟悉常见外设的硬件连接**：了解SPI、I2C、UART、单总线等接口的电气特性（上拉电阻、电平匹配）。
5. **理解锁存器与译码器在IO扩展中的应用**：掌握动态扫描原理，学会计算扫描频率和亮度关系。

### 固件编程阶段
1. **掌握STM32CubeMX的工程配置**：包括时钟树设置、外设启用、中断优先级分配、生成MDK工程。
2. **熟悉HAL库API**：熟练使用 `HAL_GPIO_*`、`HAL_SPI_*`、`HAL_I2C_*`、`HAL_UART_*` 等函数。
3. **理解SPI协议并完成LCD驱动**：学会发送命令和数据，实现字符和图形绘制（如画点、画线、显示汉字）。
4. **掌握I2C协议并完成RX8025/ADS1115驱动**：理解设备地址、寄存器读写流程，实现时间获取和ADC转换。
5. **理解单总线协议并完成DHT11驱动**：掌握微秒级延时方法，校验数据有效性。
6. **掌握中断与定时器的基本使用**：使用定时器产生中断实现数码管动态扫描，或测量脉冲宽度。

### 综合实践
1. **完成一个完整嵌入式项目的开发流程**：从需求分析、硬件设计、PCB制作、焊接调试到固件开发、系统联调，体验工程化过程。
2. **培养硬件调试与问题排查能力**：学会使用万用表、示波器、逻辑分析仪定位故障。
3. **掌握技术文档的编写规范**：撰写设计报告，包括项目背景、系统框图、硬件原理、软件流程、测试数据、心得体会。
